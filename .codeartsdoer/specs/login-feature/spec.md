# 登录功能需求规格文档

## 文档信息
- **项目名称**: MusicHome - 多设备音乐界面应用
- **功能模块**: 用户登录功能
- **版本号**: v1.0
- **创建日期**: 2026-05-19
- **最后更新**: 2026-05-19

## 1. 概述

### 1.1 功能简介
为MusicHome HarmonyOS应用添加用户登录功能，支持用户通过账号密码方式进行身份认证，实现用户身份识别和个性化服务的基础能力。

### 1.2 业务背景
当前MusicHome应用仅提供音乐浏览、播放和评论功能，缺乏用户身份识别能力。添加登录功能后，可以：
- 识别用户身份，为后续个性化推荐、收藏同步等功能奠定基础
- 保护用户数据安全，确保用户操作可追溯
- 提升用户体验，支持跨设备数据同步

### 1.3 范围界定
**包含范围**:
- 用户登录界面设计与实现
- 账号密码登录方式
- 登录状态管理
- 登录信息本地存储
- 退出登录功能
- 登录状态持久化

**不包含范围**:
- 用户注册功能（假设用户已预先注册）
- 第三方登录（如华为账号、微信等）
- 找回密码功能
- 多因素认证
- 用户信息修改功能
- 服务端API实现（假设后端接口已存在）

## 2. 用户角色

### 2.1 角色定义
| 角色名称 | 角色描述 | 权限说明 |
|---------|---------|---------|
| 未登录用户 | 未进行身份认证的用户 | 可浏览音乐内容，无法使用个性化功能 |
| 已登录用户 | 完成身份认证的用户 | 可使用所有功能，包括个性化服务 |

## 3. 功能需求

### 3.1 登录入口

#### REQ-LOGIN-001: 登录入口展示
**类型**: 功能需求
**优先级**: 高
**描述**: 系统应在应用主界面提供登录入口

**验收标准**:
- 当用户未登录时，系统应在主界面显示登录按钮或入口
- 当用户已登录时，系统应显示用户头像或用户名
- 当用户点击登录入口时，系统应跳转至登录页面

**前置条件**:
- 应用已启动并加载完成

**后置条件**:
- 登录入口状态正确显示

### 3.2 登录界面

#### REQ-LOGIN-002: 登录界面展示
**类型**: 功能需求
**优先级**: 高
**描述**: 系统应提供完整的登录界面

**验收标准**:
- 当用户进入登录页面时，系统应显示用户名输入框
- 当用户进入登录页面时，系统应显示密码输入框
- 当用户进入登录页面时，系统应显示登录按钮
- 当用户进入登录页面时，系统应显示返回按钮
- 当用户在密码输入框输入时，系统应默认以密文形式显示密码

**前置条件**:
- 用户点击登录入口

**后置条件**:
- 登录界面正确渲染

#### REQ-LOGIN-003: 输入信息验证
**类型**: 功能需求
**优先级**: 高
**描述**: 系统应验证用户输入的登录信息

**验收标准**:
- 当用户名输入框为空且用户点击登录时，系统应提示"请输入用户名"
- 当密码输入框为空且用户点击登录时，系统应提示"请输入密码"
- 当用户名格式不正确时，系统应提示"用户名格式不正确"
- 当密码长度少于6位时，系统应提示"密码长度不能少于6位"

**前置条件**:
- 用户在登录界面

**后置条件**:
- 输入验证结果正确反馈

### 3.3 登录执行

#### REQ-LOGIN-004: 登录成功处理
**类型**: 功能需求
**优先级**: 高
**描述**: 系统应正确处理用户登录成功的情况

**验收标准**:
- 当用户输入正确的用户名和密码且点击登录时，系统应调用登录接口进行验证
- 当登录接口返回成功时，系统应保存用户登录状态
- 当登录成功时，系统应保存用户基本信息（用户名、用户ID、Token）
- 当登录成功时，系统应跳转回上一页面或主页
- 当登录成功时，系统应显示"登录成功"提示信息

**前置条件**:
- 用户输入有效的用户名和密码

**后置条件**:
- 用户处于登录状态
- 用户信息已保存

#### REQ-LOGIN-005: 登录失败处理
**类型**: 功能需求
**优先级**: 高
**描述**: 系统应正确处理用户登录失败的情况

**验收标准**:
- 当用户名不存在时，系统应提示"用户不存在"
- 当密码错误时，系统应提示"密码错误"
- 当登录失败时，系统应保留用户已输入的用户名
- 当登录失败时，系统应清空密码输入框
- 当连续登录失败超过5次时，系统应限制登录尝试并提示"登录失败次数过多，请稍后再试"

**前置条件**:
- 用户输入的用户名或密码不正确

**后置条件**:
- 用户仍处于未登录状态
- 错误信息正确显示

### 3.4 登录状态管理

#### REQ-LOGIN-006: 登录状态持久化
**类型**: 功能需求
**优先级**: 高
**描述**: 系统应持久化保存用户的登录状态

**验收标准**:
- 当用户登录成功时，系统应将登录信息保存至本地存储
- 当应用重新启动时，系统应检查本地存储中的登录信息
- 当本地存在有效登录信息时，系统应自动恢复登录状态
- 当登录信息过期时，系统应清除过期信息并要求重新登录

**前置条件**:
- 用户已登录或应用启动

**后置条件**:
- 登录状态正确恢复或清除

#### REQ-LOGIN-007: 退出登录
**类型**: 功能需求
**优先级**: 中
**描述**: 系统应提供退出登录功能

**验收标准**:
- 当用户已登录时，系统应提供退出登录选项
- 当用户点击退出登录时，系统应清除本地存储的登录信息
- 当退出登录成功时，系统应更新界面显示为未登录状态
- 当退出登录成功时，系统应跳转至主页或登录页

**前置条件**:
- 用户已登录

**后置条件**:
- 用户处于未登录状态
- 登录信息已清除

### 3.5 网络异常处理

#### REQ-LOGIN-008: 网络异常处理
**类型**: 功能需求
**优先级**: 高
**描述**: 系统应正确处理网络异常情况

**验收标准**:
- 当网络不可用时，系统应提示"网络不可用，请检查网络连接"
- 当登录请求超时时，系统应提示"网络请求超时，请重试"
- 当服务器返回错误时，系统应显示相应的错误提示信息
- 当网络恢复后，系统应允许用户重新尝试登录

**前置条件**:
- 用户尝试登录

**后置条件**:
- 异常信息正确提示
- 用户可重新操作

### 3.6 界面交互

#### REQ-LOGIN-009: 密码可见性切换
**类型**: 功能需求
**优先级**: 低
**描述**: 系统应支持密码明文/密文切换显示

**验收标准**:
- 当用户点击密码可见性切换按钮时，系统应切换密码显示状态
- 当密码处于密文状态时，系统应显示密码隐藏图标
- 当密码处于明文状态时，系统应显示密码可见图标

**前置条件**:
- 用户在登录界面

**后置条件**:
- 密码显示状态正确切换

#### REQ-LOGIN-010: 记住密码功能
**类型**: 功能需求
**优先级**: 低
**描述**: 系统应支持记住密码功能

**验收标准**:
- 当用户勾选"记住密码"选项时，系统应在下次登录时自动填充用户名和密码
- 当用户取消勾选"记住密码"选项时，系统应清除已保存的密码信息
- 当应用重新打开时，系统应根据记住密码选项自动填充登录信息

**前置条件**:
- 用户在登录界面

**后置条件**:
- 记住密码功能正确执行

## 4. 非功能需求

### 4.1 性能需求
- 登录请求响应时间应不超过3秒
- 登录界面加载时间应不超过1秒
- 本地登录状态检查应不超过100毫秒

### 4.2 安全需求
- 密码在传输过程中应使用加密传输（HTTPS）
- 密码在本地存储时应使用加密存储
- 用户Token应设置合理的过期时间
- 登录失败次数限制应有效防止暴力破解

### 4.3 可用性需求
- 登录界面应支持响应式布局，适配不同设备（手机、平板、折叠屏）
- 输入框应支持软键盘自动弹出
- 错误提示信息应清晰易懂
- 界面操作应符合HarmonyOS设计规范

### 4.4 兼容性需求
- 支持HarmonyOS 5.1.0 Release及以上版本
- 支持直板机、双折叠、平板等设备形态
- 支持智能穿戴设备（可选）

## 5. 约束条件

### 5.1 技术约束
- 使用HarmonyOS ArkTS开发语言
- 遵循项目现有的三层架构（common、features、products）
- 使用项目现有的路由管理机制
- 使用Preferences进行本地数据存储

### 5.2 业务约束
- 假设后端登录API已存在并可用
- 用户账号需预先注册，本功能不包含注册流程
- 登录功能为可选功能，不影响现有音乐浏览和播放功能

## 6. 数据需求

### 6.1 输入数据
| 数据项 | 数据类型 | 必填 | 校验规则 | 说明 |
|-------|---------|-----|---------|-----|
| 用户名 | string | 是 | 长度3-20字符，仅支持字母、数字、下划线 | 用户唯一标识 |
| 密码 | string | 是 | 长度6-20字符 | 用户登录凭证 |
| 记住密码 | boolean | 否 | true/false | 是否保存登录信息 |

### 6.2 输出数据
| 数据项 | 数据类型 | 说明 |
|-------|---------|-----|
| 用户ID | string | 用户唯一标识符 |
| 用户名 | string | 用户显示名称 |
| Token | string | 用户认证令牌 |
| 登录状态 | boolean | 用户当前登录状态 |

### 6.3 本地存储数据
| 数据项 | 数据类型 | 存储方式 | 说明 |
|-------|---------|---------|-----|
| userId | string | Preferences | 用户ID |
| userName | string | Preferences | 用户名 |
| token | string | Preferences（加密） | 认证令牌 |
| loginTime | number | Preferences | 登录时间戳 |
| rememberPassword | boolean | Preferences | 是否记住密码 |
| savedPassword | string | Preferences（加密） | 保存的密码 |

## 7. 接口需求

### 7.1 用户界面
- 登录入口：在主页或个人中心显示登录按钮/用户头像
- 登录页面：包含用户名输入框、密码输入框、登录按钮、返回按钮
- 状态显示：登录成功后显示用户信息，未登录显示登录入口

### 7.2 系统接口
**登录接口**:
- 接口路径: `/api/user/login`
- 请求方法: POST
- 请求参数: `{ username: string, password: string }`
- 响应数据: `{ code: number, message: string, data: { userId: string, userName: string, token: string } }`

**Token验证接口**:
- 接口路径: `/api/user/verify`
- 请求方法: GET
- 请求头: `{ Authorization: Bearer ${token} }`
- 响应数据: `{ code: number, message: string, data: { valid: boolean } }`

## 9. 术语定义
| 术语 | 定义 |
|-----|-----|
| Token | 用户认证令牌，用于标识用户身份和会话有效性 |
| Preferences | HarmonyOS本地数据存储机制 |
| 登录状态 | 用户是否已完成身份认证的状态标识 |
| 持久化 | 将数据保存至本地存储，应用重启后仍可读取 |

- 所有需求均可转化为可执行的测试用例
